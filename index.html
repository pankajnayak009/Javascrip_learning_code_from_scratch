Introduction
what is JavaScript

JavaScript (JS) is a lightweight interpreted (or just-in-time compiled) programming language with first-class functions. While it is most well-known as the scripting language for Web pages, many non-browser environments also use it, such as Node.js, Apache CouchDB and Adobe Acrobat. JavaScript is a prototype-based, multi-paradigm, single-threaded, dynamic language, supporting object-oriented, imperative, and declarative (e.g. functional programming) styles.

Loops and Strings

JavaScript Loops (Notes & Table)

Loops in JavaScript help execute a block of code multiple times until a specified condition is met. They are used for iteration, automation, and efficiency in programming.

Types of Loops in JavaScript:

  • for loop → Runs a block of code a fixed number of times.
  • while loop → Runs as long as a condition is true.
  • do...while loop → Runs at least once, then checks the condition.
  • for...in loop → Loops over object properties (keys).
  • for...of loop → Loops over iterable items (arrays, strings, etc.).
  • break statement → Exits a loop immediately.
  • continue statement → Skips the current iteration and moves to the next.

Call Stack and Hoisting
Hoisting in JavaScript

Hoisting is a behavior in JavaScript where function and variable declarations are moved to the top of their scope before execution.

How It Works
  • Function Declarations are fully hoisted, meaning they can be called before being defined.
  • Var Variables are hoisted, but only the declaration (not the initialization) let a; Declaration , a = 50;Initialization
  • Let and Const Variables are hoisted, but they are in a TDZ (Temporal Dead Zone) until initialized
Call Stack and Hoisting
Call Stack in JavaScript

The call stack is a data structure in JavaScript that keeps track of function calls. It follows the Last In, First Out (LIFO) principle.

How It Works
  • first() is pushed onto the stack.
  • first() calls Second(), so Second() is pushed onto the stack.
  • Second() calls Third(), so Third() is pushed onto the stack.
  • Third() executes and is popped.
  • Second() finishes and is popped.
  • first() finishes and is popped.
  • The Call Stack is empty. ✅

Variable Scop in JavaScript

Variable Scope. Scope in JavaScript refers to the current context of code, which determines the accessibility of variables to JavaScript. The two types of scope are local and global: Global variables are those declared outside of a block. Local variables are those declared inside of a block.24

Types of Scop
  • Global Scop ex : var
  • Function Scop
  • Block Scop ex : let , const

    • Note : var is not block Scope

Temporal Dead Zone (TDZ) in JavaScript

What is the Temporal Dead Zone? The Temporal Dead Zone refers to the period where a variable exists in a scope but cannot be accessed until it is initialized. The TDZ starts from the beginning of the block until the variable is declared and initialized.

Classes in JavaScript
Classes in JavaScript

JavaScript classes provide a structured way to create objects using the class keyword. They were introduced in ES6 to make object-oriented programming (OOP) easier in JavaScript.

Key Features of Classes

  • Encapsulation Groups data and methods into a single unit.
  • Inheritance: Allows a class to inherit properties and methods from another class.
  • Reusability: Promotes code reuse by defining reusable object blueprints.
  • Constructor: A special method that initializes object properties when a class is instantiated.
  • Methods (Behaviors): Functions within a class that define the actions an object can perform.
  • Getters & Setters:
    • Getter: A method that retrieves a private property.
    • Setter: A method that modifies a private property.
  • Function: printEngineerName: A function that prints the engineer’s name, using a default value if no argument is provided.

DOM Manipulation in js

JavaScript DOM (Document Object Model) Notes

1. What is the DOM?

The DOM (Document Object Model) is a programming interface that allows JavaScript to interact with and manipulate HTML and CSS.

It represents the HTML document as a tree structure, where each element is a node.

2.Types of Nodes in the DOM

  • Element Nodes: Represent HTML elements (<p>, <div>, <h1>, etc.).
  • Text Nodes: Contain text inside an element.
  • Attribute Nodes: Represent attributes of elements (class, id, src, etc.).
  • Comment Nodes: Represent HTML comments (<!-- This is a comment -->).

3. Accessing Elements in the DOM

document.getElementById("id");
document.getElementsByClassName("class");
document.getElementsByTagName("tag");
document.querySelector("selector");
document.querySelectorAll("selector");

4. Modifying the DOM

Changing Content

document.getElementById("heading").textContent = "New Title";
document.getElementById("content").innerHTML = "Bold Text";

Changing Attributes

let img = document.getElementById("myImage");
img.setAttribute("src", "new-image.jpg");
img.removeAttribute("alt");

Changing Styles

let box = document.getElementById("box");
box.style.backgroundColor = "blue";
box.classList.add("active");

5. Creating and Removing Elements

Creating Elements

let newPara = document.createElement("p");
newPara.textContent = "This is a new paragraph.";
document.body.appendChild(newPara);

Removing Elements

let item = document.getElementById("item");
item.remove();

6. Event Handling in the DOM

document.getElementById("btn").addEventListener("click", function() {
    alert("Button clicked!");
